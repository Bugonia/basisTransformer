# Claim-Evidence Map

## Main Claim

Claim:

> Direct residual write access by both Attention and FFN basis families is an
> architectural resource in Transformer language models.

Evidence:

- `standard` outperforms block variants where only one module writes directly.
- `standard_fa` remains much closer to `standard` because it preserves dual
  writes despite reversing order.
- `parallel` is intermediate because it preserves dual writes but weakens
  same-layer coefficient coupling.
- carry variants preserve additional coefficient modulation but remain far below
  standard dual-write blocks.

Status:

- supported by existing enwik8 experiments;
- needs stronger statistical package and open-model diagnostic validation.

## Claim 1: Attention Has a Direct Write Basis

Claim:

> Attention writes to the residual stream through columns of the output
> projection, with coefficients formed from attention routing and value
> coordinates.

Evidence:

- mathematical decomposition in the technical report;
- direct mapping to GPT-2/Pythia/Qwen projection modules;
- planned basis inventory on pretrained models.

Status:

- mathematically supported;
- needs model inventory result table.

## Claim 2: FFN Has a Direct Write Basis

Claim:

> FFN writes through columns of its output/down projection, with coefficients
> generated by nonlinear hidden activations.

Evidence:

- standard FFN decomposition;
- module path inventory in pretrained models;
- planned coefficient logging.

Status:

- mathematically supported;
- needs open-model inventory table.

## Claim 3: Coefficient Modulation Does Not Substitute for Direct Write Access

Claim:

> Letting a module influence another module's coefficients does not fully
> substitute for letting that module write its own basis directions directly.

Evidence:

- `block_af_carry` and `block_fa_carry` give additional carry signals but remain
  much weaker than `standard`;
- topology sweep shows carry can help some block variants, but does not close
  the gap to standard dual-write blocks.

Status:

- supported;
- include both AdamW/30k and Muon/100k settings if space permits.

## Claim 4: Output Projections Matter Differently When They Are Write Outlets

Claim:

> An output projection can be algebraically absorbable when it is internal to a
> block-composed pathway, but structurally meaningful when it is the final
> residual write outlet.

Evidence:

- `block_af_no_mid_ln_no_wo` remains close to or slightly better than the
  no-middle-LN variant with `W_O`;
- the original `block_af` with middle LN remains stronger;
- standard Transformer needs `W_O` as the Attention write basis outlet.

Status:

- supported as control evidence;
- present carefully to avoid confusing reviewers.

## Claim 5: Pretrained Open Models Expose the Same Write-Basis Structure

Claim:

> Mainstream open decoder-only models instantiate the same architectural basis
> families, enabling basis-level attribution and intervention.

Evidence:

- planned basis inventory for Pythia-70M, GPT-2, Qwen2.5-0.5B;
- planned logit attribution and ablation.

Status:

- not yet complete;
- highest-priority new experiment for submission credibility.

## Risk Claims to Avoid

Avoid claiming:

- all basis directions are interpretable or monosemantic;
- the framework fully explains all Transformer performance;
- results already prove scaling laws;
- enwik8 character-level results alone settle large LLM architecture design;
- activation steering, SAE, compression, and hallucination applications are
  already solved by this paper.
